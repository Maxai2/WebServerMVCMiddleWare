
Протокол HTTP (HyperText Transfer Protocol).


HTTP - это протокол прикладного уровня (7ой уровеньв модели OSI).

Изначально протокол HTTP был предназначен для передачи гипертекстовых документов 
(то есть документов, которые могут содержать ссылки, позволяющие организовать 
переход к другим документам).

На сегодня HTTP - широко распространённый протокол передачи любых данных (text, html,
xml, json...) от сервера к клиенту и наоборот (обычно это веб-сервер и браузер).

Протокол HTTP предполагает использование клиент-серверной структуры передачи данных. 
Клиентское приложение формирует запрос и отправляет его на сервер, после чего 
серверное программное обеспечение обрабатывает данный запрос, формирует ответ и 
передаёт его обратно клиенту. После этого клиентское приложение может продолжить 
отправлять другие запросы, которые будут обработаны аналогичным образом.


Запросы, которые идут по http протоколу, состоят из таких частей:
+ method (GET, POST, PUT, DELETE)
+ URI (путь к конкретному ресурсу)
+ version (HTTP/1.1)
+ headers (обязательный host - куда запрос, все заголовки - 
				https://developer.mozilla.org/ru/docs/Web/HTTP/Заголовки)
+ body (дополнительные пользовательские данные)

Ответы имеют такую же структуры, но к ним добавляется еще:
+ status code (три цифры, которые определяют результат совершения запроса) 
+ reason phrase (текстовое пояснение к коду ответа)
